Cats like boxes... web devs too!

The common denominator between cats and web devs? They both like boxes of course!

Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

I liked Udacity's analogy comparing a webpage to a house with a structure with different rooms that have different functions, that are decorated, and elements that can be interacted with. I'll give a go at creating my own analogy.

My analogy is to a couple dancing choreography. This explanation might not work with everyone, but I have lots of dancer friends!

Couple dances are usually structured in patterns or "moves", elements that gets reused to build the structure of your dance. Patterns would be our HTML tags, and our choreography (combination of patterns) is our entire HTML.

You can add styling to every dance pattern: a hand movement, a knee up, highs and lows. This is our CSS.

Have you ever looked at a couple dancing while sticking purely to patterns without styling? It is very boring, much like HTML without any CSS!

Dancers will need to store and make sense of the combination of patterns in a way that their brain can handle. This is the DOM.

The browser is giving a visual representation of the DOM - based on the HTML/CSS, in the same way the dancers are rendering the choreography stored in their brain. Thankfully, browsers are more consistent than dancers and less prone to mistakes!

If you've never done couple dancing in your life, and you're wondering what the heck I'm talking about, I apologize. I'm willing to give dance classes to the whole cohort to make it up!

What is meant by boxifying design?

Every element on a web page is contained in a rectangle, in other words a box. Boxifying design means taking every element of a webpage design and dividing it into boxes. A box will often include other boxes. For example, the overall page is a box. This box in turn can include two boxes: a main title, and a blog article, which in turn might include a box for an image and a box for text. This also applies to a button with rounded corners, or any other element that does not visually look rectangular, it's a box!

The outline css property can be used to make the boxes apparent on a webpage. I find it very useful both for debugging my own css and for understanding how an existing webpage is structured.

What is the box model?

Knowing that everything in a webpage is a box, we can define several areas within and outside this box:

  • content: the actual content inside the box, e.g. text
  • padding: the space between the content and the border of the box
  • border: the border of the box
  • margin: the space between this box and other boxes

Padding, border and margin values can be set through css properties.

There are several ways to compute the size (width and height css properties) of a box.You can choose which method to use through the box-sizing property. Here are the ones I know so far:

  • content-box: This is the default value for the box-sizing property. This means width and height will include only the content of the box. padding, border and margin are excluded
  • border-box: This means width and height include the content, padding and border, but not the margin.

I find border-box practical as it allows you to define your width with a percentage, and add padding afterwards without having to change the percentages (as the padding is included in the size of the box).

The box model for an element along with its content, padding, border and margin width is displayed in the web dev tools when inspecting an element!
