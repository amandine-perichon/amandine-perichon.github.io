JavaScript Pony Language

Use cases for parentheses ( )

Parentheses are used to define a function. The parentheses will contain zero, one or more parameters 


            function hello () {
              console.log("Hello")
            }
          

            function helloPony (pony) {
              console.log("Hello " + pony)
            }

Parentheses are used to call a function. The parentheses will contain zero, one or more arguments 

hello()
helloPony("Twilight Sparkle")

Parentheses can contain an anonymous function (module) which will be called right away 


            (function () {
              var myPony = "Rainbow Dash"
              helloPony(myPony)
            })()

Parentheses are used to group expressions and force a certain order between operations 


            var poniesOnScreen = (3 + 4) * 2

Parentheses contain a boolean expression which will be evaluated in an if, while, do/while control flow or loop 


            if (myFavoritePony === "Pinkie Pie") {
              console.log("I'm gonna make you smile and I will brighten up your day-aaay!")
            }

Parentheses contain the initilisation, end condition and increment statements in a for loop 


            for (var i = 0; i < ponies.length; i++) {
              console.log(i)
            }

Parentheses contain a case parameter in a switch statement 


            switch (color) {
              case "pink":
                myPony = "Pinkie Pie"
                break
              case "green":
                myPony = "AppleJack"
                break
              case "rainbow"
                myPony = "Rainbow Dash"
            }

Parentheses can be used as... parentheses in a string 

console.log("These are parentheses: ( )")

Use cases for brackets [ ]

Brackets are used to create an array literal 


            var LittlePonies = ["Twilight Sparkle", "Rainbow Dash", "Pinkie Pie"]

Brackets are used to to access an array element, through the bracket notation 


            myFavoritePony = LittlePonies[2]

Brackets are used in regular expressions to represent a group or range of characters 


          var hasKorP = /[kp]/
            console.log(("Pinkie Pie").match(hasKorP))

Brackets can be used as... brackets in a string 

console.log("These are brackets: [ ]")

Use cases for braces { }

Braces are used to create an object literal 


            var PinkiePie = {
              name: "Pinkie Pie",
              gender: "Female",
              favouriteColor: "pink"
            }

Braces are used to group several statements in a block, for instance in a control flow, or to define a function 


            if (myFavoritePony === "Pinkie Pie") {
              console.log("I'm gonna make you smile and I will brighten up your day-aaay!")
            }
          

            function helloPony (pony) {
              console.log("Hello " + pony)
            }

Use cases for single quotes ' ' and double quotes " "

Single and double quotes are used to create a string literal 

var ponyName = "Applejack"
var ponyName2 = 'Mrs. Cup Cake'

Double quotes can be used to eliminate the need to escape apostrophes 

var ponyLine = "Hey Twilight Sparkle, have you seen Applejack's new hat?"

Single quotes can be used to eliminate the need to escape double quotes (and vice versa) 

var anotherPonyLine = 'Then, Rarity said: "Oh, Fluttershy? Could you be a dear and fly faster please."'

It also looks like JSONs should always include double quotes, not single quotes. But the ponies aren't too worried about this yet.
