Implementing a first responsive website

What a responsive site is, and why responsiveness is important

A responsive website resizes and adapts to different sizes of screen (different resolutions) and types of devices, ensuring the website is still legible and is looking great.

There is a multitude of devices out there - desktops, laptops, phones, tablets of different sizes. Responsiveness is important to ensure great user experience on all of these devices.

What mobile first design is, and why it's important

Mobile first design is the process of designing your website firstly for mobile devices. Your first wireframes will be for mobile screens, and your basic CSS rules will be written to make the content look legible and good on mobile. CSS rules for desktop only come in a second phase of your development process and are included in media queries to be applied only on screen sizes above a certain threshold, also called breakpoint.

Mobile first design is important as is it easier to design for small screens then scale up, than the other way around. It helps focus on the content legiblibility and relevence rather than focusing on the design. Overall, it leads to better user experience.

What frameworks are, and their pros and cons

A framework is a predefine structure, or basic solution to a common problem. For instance, the CSS Skeleton framework includes a grid system for building websites. Rather than each web developer recreating a grid structure everytime they build a website, they can use an existing framwork and don't have to create a grid system. There are different frameworks solving the same standard problem, with solutions having different complexities and fitting different instances of that problem.

Advantages to using a framwork:

  • Efficiency: unsing a framework is easier and faster, you don't need to reinvent the wheel!
  • Good practices: frameworks are written using best practices and standards, e.g. with regards to breakpoints for media queries
  • Testing: frameworks consider browser compatibility which is not easy to test, and quality is high thanks to the feedback from the developer community
  • Collaboration: all project members use the standard and naming conventions set by the framework which make collaboration easier
Disadvantages to using a framework:
  • Learning curve: Learning to use a framework can be quite time consuming
  • Customization: framworks can be limiting, and require a lot of customization. This can represent a lengthy addition of code in order to override framework styles which become unused code
  • Mastery: using a framework, a developer, especially an inexperienced one, misses out on learning to implement the basics, and ends up not knowing the code as well as if they had written it

What a wireframe is and why we use it

A wireframe is a static picture, a mockup of what your website layout will look like. It helps have an overview of the structure of the website, its skeleton, amd make sure all content elements are included.

Wireframes can be deconstructed into boxes (boxifying), which will be the basic elements of our HTML. Basic elements of design are also included, for example: number of columns, formart of the menu. Wireframes usually do not include colors and images, rather they use simple lines - wires - to represent the elements on the page.
